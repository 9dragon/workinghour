# 中间件组件使用指南

在数字工厂 / 项目管理系统中，各中间件的定位和典型用途。

---

## MySQL

**定位**：关系型主数据库，存储业务核心数据。

典型存储内容：
- 用户、组织、角色、权限
- 设备档案（静态信息）
- 工单、生产计划、物料 BOM
- 项目、任务、里程碑
- 审计日志、系统配置

**不适合存**：设备实时时序数据、日志、非结构化配置

---

## Redis

**定位**：缓存层 + 轻量状态存储。

典型使用场景：

| 场景 | 说明 |
|------|------|
| 会话/Token | JWT 黑名单、Refresh Token 存储 |
| 权限缓存 | 用户角色权限，TTL 5min，变更时主动失效 |
| 设备在线状态 | 设备最新状态快照，高频读取 |
| 分布式锁 | 工单并发操作防重（Redisson） |
| 限流计数 | 接口访问频率控制 |
| 字典/配置缓存 | 低频变更的基础数据 |

**注意**：不持久化重要业务数据；Redis 宕机不影响核心功能。

---

## MongoDB

**定位**：时序数据 + 非结构化数据存储。

典型存储内容：
- 设备采集指标（温度、压力、电流等时序数据）
- 设备告警历史
- 操作日志 / 行为轨迹
- 工单执行过程记录（附带动态扩展字段）

**索引必建**：
- `{ deviceId: 1, timestamp: -1 }` 用于设备数据查询
- TTL 索引控制历史数据自动清理（如保留 90 天）

---

## EMQX

**定位**：工业设备 MQTT 消息接入层。

典型架构：
```
PLC/传感器 → MQTT → EMQX → 规则引擎 → Spring Boot 服务
```

关键配置：
- 规则引擎：按 topic 过滤并转发到后端（Webhook / RabbitMQ bridge）
- QoS：设备遥测用 QoS 0（性能优先），控制指令用 QoS 1（可靠性优先）
- 认证：X.509 证书认证（禁匿名），按设备 ID 隔离 topic

Topic 命名规范：
```
设备上报：device/{tenantId}/{deviceId}/telemetry
控制指令：device/{tenantId}/{deviceId}/command
设备状态：device/{tenantId}/{deviceId}/status
```

---

## RabbitMQ

**定位**：业务异步解耦 + 任务队列。

典型使用场景：

| Exchange / Queue | 用途 |
|-----------------|------|
| `work-order.exchange` | 工单状态变更事件（审批通过、完工等） |
| `report.task.queue` | 报表异步生成任务 |
| `notification.queue` | 消息通知（站内信、短信触发） |
| `device.alert.queue` | 设备告警处理队列（防止告警风暴压垮主服务） |

**可靠性保障**：
- 生产者：开启 Confirm 机制
- 消费者：手动 ACK，处理完成后确认
- 死信队列：消费失败 3 次后转 DLQ，人工介入
- 消息幂等：业务层用唯一 messageId + Redis 去重

## 组件选型决策表

| 需求场景 | 推荐组件 |
|---------|---------|
| 关系数据 + 事务 | MySQL |
| 设备实时状态读取 | Redis（缓存） |
| 设备历史数据查询 | MongoDB |
| 设备数据接入 | EMQX |
| 异步任务 / 事件驱动 | RabbitMQ |
| 文件存储 | OSS/MinIO（不用数据库） |
| 全文搜索（如有） | Elasticsearch（按需引入） |
